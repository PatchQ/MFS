## 1. 一句話總結

MFS 是一個**以「技術指標信號 + 機器學習模型」雙引擎為核心**、透過**回測框架（`backtesting.py`）驗證策略**，並搭配**多套 Web 工具**（HERMES 語音助手 / so_viewer 期權查看器 / hk_stock_gann_scanner 掃描器）的港股量化研究平台。資料流覆蓋「下載 → 清洗 → 指標 → 訊號 → 回測 → AI 預測 → 信號代理 → 視覺化」全鏈路。

## 4. 核心架構（新版 — 5 大新模組）

### 4.1 `Core/MFSDataHub.py` — 單例數據中心 + 事件總線

- **單例模式**（threading.Lock 保護）
- 功能：`set_data(key, df)` / `get_data(key)` / `subscribe(event_types, callback)` / `publish(event)` / `register_indicator(name, cls)`
- 事件類型：`DATA_UPDATED`, `INDICATOR_REGISTERED`, `INDICATOR_COMPUTED`, `MARKET_ALERT`, `CUSTOM`
- 向後兼容：`set_ohlcv(symbol, timeframe, df)` / `get_ohlcv(symbol, timeframe)`（讓舊 TA 模組仍能用）

### 4.2 `Core/IndicatorEngine.py` — 指標準執行引擎

- 自動**拓撲排序**（Kahn 算法）解析指標依賴
- **緩存**（`(data_key, indicator_name, params_hash)` → `IndicatorResult`，TTL 300s）
- **多周期合併**：`compute_multi_timeframe(symbol, timeframes, ...)`
- 向後兼容：`run_indicator(df, name, params, **kwargs)` / `add_indicator(df, name, ...)`
- 註冊 7 個 Wrapper：`IchimokuWrapper`, `HFHWrapper`, `BossWrapper`, `VCPWrapper`, `FisherWrapper`, `GBS22CWrapper`, `Breakout200Wrapper`

### 4.3 `DataSource/` — 數據源插件系統

- 抽象基類 `BaseDataSource`（`name` / `get_ohlcv` / `is_available`）
- `DataSourceRegistry` 單例，`get_ohlcv(ticker)` 自動選擇合適源
- 已註冊：`YahooFinanceSource`, `HKEXSource`, `TencentSource`
- 已用於 `UTIL/LW_Collect.py` 的 `DS_YFgetAll()` 取代舊的 `YFgetAll()`

### 4.4 `ScriptRunner/` — Subprocess + JSON-line 調度器

- `ScriptRunner("python ProcessTA.py --json-stdin", max_workers=4)`
- 流程：每個請求 spawn subprocess → stdin 寫 JSON → stdout 收 JSON response
- 適合在 `ProcessTA.py` / `ProcessAI.py` 仍維持模組級共享 `IndicatorEngine` 實例的前提下做併發隔離

### 4.5 `SignalAgent/` — AI 信號代理

- `SignalMemory`：SQLite 持久化（`signals.db`），索引 `(ticker, date)`、`model`
- `SignalPredictor`：載入 `Model/{MODEL_NAME}/P_{ticker}.pkl`，過濾 `DROP_COLS`，呼叫 `predict_proba`
- `SignalReasoner`：規則推理（RSI 超買/超賣、MACD 金叉/死叉、一目均衡表雲層）
- `SignalOutput`：格式化輸出
